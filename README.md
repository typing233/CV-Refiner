# CV-Refiner 简历评估工作流

基于 Dify 平台的智能简历评估工作流，从结构完整性、语言表达、关键词覆盖、量化成果、冗余内容五个维度对简历进行全面评估，输出可直接阅读的 Markdown 格式评估报告。

## 功能特点

- **五维评估体系**：结构完整性、语言表达、关键词覆盖、量化成果、冗余内容
- **结构化输出**：每条问题附带负面影响说明和具体修改建议
- **评分系统**：总评分 + 各维度独立评分，直观量化简历质量
- **双输入模式**：支持粘贴文本或上传文件（txt/md/pdf/docx），文件自动提取文本
- **条件分支**：校验失败时直接返回错误信息，不调用大模型，节省 token
- **输入校验**：自动检测无效输入、内容过短/过长、非简历文本
- **错误降级**：LLM 输出异常时自动降级为原始文本展示

## 工作流结构

```
                                                          ┌─ valid=true ──→ [简历评估分析 GPT-4o] → [报告格式化] → [输出报告]
[开始] → [文档提取] → [输入合并] → [输入校验] → [条件分支]─┤
                                                          └─ valid=false ─→ [错误输出]
```

- **文档提取**：从上传文件中提取纯文本（支持 txt/md/pdf/docx）
- **输入合并**：优先使用文件提取内容，其次使用粘贴文本
- **条件分支**：IF/ELSE 节点，valid=true 才进入 LLM，无效输入不消耗 token

## 快速开始

### 前置要求

- Dify 平台（v0.6.0+），本地自部署或 Dify Cloud
- OpenAI API Key（需支持 GPT-4o 模型）

### 导入步骤

1. **部署 Dify**（如已有实例可跳过）

   ```bash
   git clone https://github.com/langgenius/dify.git
   cd dify/docker
   cp .env.example .env
   docker compose up -d
   ```

   访问 `http://localhost/install` 完成初始化设置。

2. **配置模型供应商**

   进入 Dify 后台 → 设置 → 模型供应商 → 添加 OpenAI → 填入 API Key。

3. **导入工作流**

   进入 Dify 后台 → 工作室 → 创建应用 → 导入 DSL → 选择 `cv_refiner_workflow.yml` 文件。

4. **测试运行**

   在工作流编辑器中点击"运行"，将 `examples/sample_resume.txt` 的内容粘贴到 `resume_text` 输入框（或直接上传该文件），点击执行。

## 输入说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| resume_text | 文本 | 可选 | 简历全文（粘贴输入），支持中英文，100~20000 字符 |
| resume_file | 文件 | 可选 | 上传简历文件，支持 .txt / .md / .pdf / .docx |

> 两者至少提供一项。同时提供时优先使用上传文件的内容。

## 输出示例

工作流输出为 Markdown 格式评估报告，包含：

- 综合评分（0-100）
- 各维度评分表格
- 简历优点列表
- 按严重程度排序的问题清单（含负面影响和修改建议）
- 优先修改事项

## 评估维度详解

| 维度 | 评估内容 |
|------|----------|
| 结构完整性 | 核心模块是否齐全（个人信息、教育、经历、技能等） |
| 语言表达 | 用词是否专业简洁、语法正确、动词有力 |
| 关键词覆盖 | 行业术语、技术栈是否充分体现 |
| 量化成果 | 工作成果是否有数据支撑（百分比、数量、时间等） |
| 冗余内容 | 是否存在无关信息、重复表述、过长描述 |

## 自定义修改

### 调整模型

在工作流编辑器中，点击"简历评估分析"节点，可更换为其他模型（如 GPT-4-turbo、Claude 等），需确保 Dify 中已配置对应模型供应商。

### 调整评估标准

编辑"简历评估分析"节点的 System Prompt，可自定义评估维度、评分标准和输出格式。

### 调整校验规则

编辑"输入校验"节点的 Python 代码，可修改字符长度限制、关键词检测逻辑等。

## 常见问题

**Q: 导入后提示模型不可用？**
A: 请确保在 Dify 设置中正确配置了 OpenAI 模型供应商并填入有效 API Key。

**Q: 评估结果输出为原始 JSON？**
A: 检查"报告格式化"节点是否正确连接到 LLM 节点的输出。降级模式下会展示原始文本。

**Q: 支持 PDF 简历吗？**
A: 支持。工作流内置文档提取节点，可直接上传 .txt / .md / .pdf / .docx 文件，自动提取文本进行评估。
